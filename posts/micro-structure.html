微觀市場結構：從掛單與吃單看清楚每一筆交易

市場裡有兩種人：「排隊等候」的限價單，和「直接插隊」的市價單。搞懂這兩種人，就搞懂了價格為什麼會動。

chillzon 2025 年 8 月 微觀結構 · 限價單 · 市價單 · 盤口

兩種人，兩種單

想像一家很熱門的拉麵店，每天都要排隊。

第一種人（限價單）：乖乖排隊，等待入座。 他們願意等，但有條件——「如果等超過 30 分鐘我就走了」，或「我只願意坐靠窗的位子」。 他們是被動的，等待條件被滿足。

第二種人（市價單）：直接走進去告訴服務生「現在有什麼位子就給我什麼位子，我馬上要坐」。 他們是主動的，願意付出「不一定最好的位置」的代價，換取「立即入座」。

限價單（Limit Order） = 排隊等候，被動，提供流動性
市價單（Market Order） = 直接插隊，主動，消耗流動性

價格移動，是因為市價單在消耗限價單。

盤口（Order Book）是什麼

盤口就是那家拉麵店的「等候名單」——列出了所有正在等待的限價單。 

盤口長這樣：

Ask（賣方掛單，想賣出的人）
  52,500  │  ████ 400 張
  52,400  │  ██████ 600 張
  52,300  │  ████████ 850 張   ← 賣方掛單越來越厚
  ─────────────────────────────  ← 現在的市場價格（約 52,250）
  52,200  │  ████████ 820 張   ← 買方掛單越來越厚
  52,100  │  ██████ 580 張
  52,000  │  ████ 420 張

Bid（買方掛單，想買入的人）

Ask 側：想賣出的人，把限價賣單掛在比現在高的價格等著。 Bid 側：想買入的人，把限價買單掛在比現在低的價格等著。

當有人下市價買入單，它會直接「吃掉」最便宜的 Ask 掛單，成交後價格往上移。 當有人下市價賣出單，它會直接「吃掉」最高的 Bid 掛單，成交後價格往下移。

誰在移動價格？

很多人覺得「很多人買所以漲」——這是對的，但不夠精確。

更準確的說法是：有人用市價單積極地去買，吃掉 Ask 側的限價賣單，才會讓價格往上走。

如果所有人都掛限價單等著，沒有人用市價單，那麼價格永遠不會動。 是市價單在消耗限價單，才讓價格移動。 

價格往上走的過程：

市價買入單出現
→ 吃掉 52,300 的 Ask 掛單（850 張）
→ 下一個 Ask 是 52,400（600 張）
→ 繼續有市價買入，吃掉 52,400
→ 價格移動到 52,400
→ 如此往上推進

流動性：市場的「緩衝墊」

掛在盤口的限價單，有個專有名詞叫做「流動性（Liquidity）」。

流動性越高 = 掛單越多 = 需要更大的市價單力量才能推動價格。 就像一塊厚厚的海綿，你要用很大的力氣才能把它壓下去。

流動性越低 = 掛單越少 = 很小的市價單就能讓價格大幅移動。 就像一塊薄薄的紙板，輕輕一按就凹進去了。

這就是為什麼在 Volume Profile 的 LVN（低成交量節點）附近， 價格移動特別快——那個區域歷史上掛單就很少，流動性薄，很容易被穿透。

止損單：被市場「順路帶走」的人

止損單是一種特殊的市價單：「當價格到達 X，自動幫我執行市價單」。

比喻：你搭捷運，下車前設了鬧鐘「到站就叫我」。鬧鐘響了，你不管有沒有座位，立刻站起來衝向門口。 這就是止損單——條件一觸發，不管當下市況，立刻執行。 

做空的人設的止損單：
  └── 在高於進場價的某個位置，掛著「市價買入」的止損單
  └── 當價格漲到那裡，自動觸發 → 變成市價買入 → 打到 Ask 側
  └── 大量止損買單被掃到 → Ask 量暴增 → 看起來像「買方積極進場」
  └── 但其實只是做空的人被迫認賠

做多的人設的止損單：
  └── 在低於進場價的某個位置，掛著「市價賣出」的止損單
  └── 當價格跌到那裡，自動觸發 → 變成市價賣出 → 打到 Bid 側
  └── 大量止損賣單被掃到 → Bid 量暴增 → 看起來像「賣方積極進場」
  └── 但其實只是做多的人被迫認賠

流動性獵殺（Liquidity Hunt）

大型的市場參與者知道：在關鍵價格附近（前高、前低、整數價位）， 有大量的止損單在等待被觸發。

這些等待被觸發的止損單，對大戶來說是「流動性」——它們一旦觸發，就會產生大量市價單，讓大戶能夠在那個位置輕鬆成交大量訂單。

所以市場有時候會故意走到那些止損密集的區域，把止損單掃光（獵殺流動性），然後反轉。 

常見場景：
  前高在 53,000
  → 很多做空的人把止損掛在 53,100（前高上方）
  → 市場拉到 53,050，掃掉這些止損（大量市價買單湧入）
  → 足跡圖看到：Ask 側暴量，但價格沒有繼續漲
  → 大戶用這些買單作為對手盤，在高點建立空單
  → 隨後反轉下跌
這就是「假突破」的微觀結構真相——不是隨機的，而是有計畫的流動性獵殺。 看懂這個，你就不會再被假突破騙進場。

冰山單（Iceberg Order）

大型機構不會把幾千張的單子全部掛在盤口上——那樣太顯眼了，市場會知道他們要幹嘛，反而對他們不利。

所以他們用冰山單：每次只在盤口顯示一小部分，成交後自動補上新的。 就像冰山一樣，你只看到水面上的小角，水下還有巨大的量在等著。

怎麼識別冰山單？看足跡圖上某個價格節點的成交量： 

如果一個價格節點：
  成交量非常大（例如 5,000 張）
  但你在下單前看盤口，那個節點只掛了 50 張
  → 意味著有大量的單子在「自動補充」
  → 這就是冰山單的痕跡
  → 代表有機構在那個價格悄悄建大倉

整合：微觀結構的完整思維

市場移動的完整邏輯：

1. 盤口裡有大量限價單（流動性）在等著
2. 市價單出現，消耗限價單，價格移動
3. 止損單在關鍵位置等著，一旦觸發也變成市價單
4. 大型參與者知道止損在哪裡，故意把價格推過去獵殺
5. 用獵殺的流動性建立自己的倉位
6. 然後往真實方向走

你要問的問題：
  「這裡的成交是真實的？還是止損被掃？」
  「Ask/Bid 哪邊是主動的？哪邊是被動的？」
  「這個位置的流動性是薄的還是厚的？」
微觀結構不是神秘的黑魔法，它就是把「誰在做什麼、為什麼做」搞清楚。 每次看到足跡圖上的數字，問自己：這筆錢是主動進場的？還是被逼出來的？ 這個問題的答案，往往就是下一步方向的線索。
